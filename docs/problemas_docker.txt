# Análisis de la transición técnica a Julia 1.11.3

## Superación de la cirugía de librerías y conflictos de grafos

La intervención técnica previa, denominada "cirugía de OpenSSL", consistía en la creación de vínculos simbólicos manuales para forzar a Julia 1.10 a utilizar las librerías criptográficas del sistema operativo (`libssl.so.3` y `libcrypto.so.3`). Si bien esta medida permitía la carga de `ArchGDAL`, generaba una inestabilidad crítica en el *linker* dinámico al ser invocado desde R. Esta discrepancia de símbolos era el origen de los fallos en `JuliaConnectoR`, el cual requiere un árbol de dependencias coherente para establecer los *sockets* de comunicación.

Adicionalmente, el error `KeyError: key "Graphs" not found` respondía a una falla estructural en el gestor de paquetes de la versión 1.10 al procesar extensiones (*weak dependencies*). La actualización a Julia 1.11.3 resuelve ambos frentes: incorpora artefactos binarios (*JLL*) compatibles de forma nativa con Ubuntu 24.04 (Noble) y utiliza un motor de resolución de metadatos rediseñado que elimina las colisiones en el archivo de manifiesto.

---

## Procedimiento de restauración técnica consolidado

El siguiente bloque de comandos realiza la migración completa del núcleo, la purga de registros corruptos y la instalación de la suite geoespacial avanzada, incluyendo soporte para cubos de datos (*YAXArrays*) y visualización interactiva para entornos de servidor (*WGLMakie*).

````bash
# 1. Sustitución de binarios a la versión estable actual (v1.11.3)
cd /tmp
rm -rf /opt/julia
wget https://julialang-s3.julialang.org/bin/linux/x64/1.11/julia-1.11.3-linux-x86_64.tar.gz
tar -xvzf julia-1.11.3-linux-x86_64.tar.gz
mv julia-1.11.3 /opt/julia

# 2. Configuración de entorno y preferencias geoespaciales
rm -rf /root/.julia/environments/v1.11
mkdir -p /root/.julia/environments/v1.11
printf "[LocalPreferences]\nGDAL_jll = { libgdal_path = \"/usr/lib/libgdal.so\" }\nGEOS_jll = { libgeos_path = \"/usr/lib/x86_64-linux-gnu/libgeos_c.so\" }\n" > /root/.julia/environments/v1.11/LocalPreferences.toml

# 3. Instalación masiva de la suite técnica (Análisis + Visualización)
julia -e '
using Pkg
ENV["JULIA_PKG_PRECOMPILE_AUTO"] = 0
Pkg.Registry.update()

# Matriz completa: Core, Grafos, Cubos de datos y Visualización Web
paquetes = [
    "Preferences", "Suppressor", "RCall", "LibGEOS", "Tables", "DataFrames", 
    "Plots", "Statistics", "ArchGDAL", "LibPQ", "GeoDataFrames", "IJulia", 
    "CSV", "CairoMakie", "WGLMakie", "AlgebraOfGraphics", "DimensionalData", 
    "FlexiJoins", "GeoFormatTypes", "GeoInterface", "GeoJSON", "GeoMakie", 
    "GeometryOps", "Makie", "MakieCore", "NaturalEarth", "Proj", "Rasters", 
    "StatsBase", "Tyler", "GeoStats", "Graphs", "NCDatasets", "MetaGraphsNext", 
    "Crayons", "GeoInterfaceRecipes", "YAXArrays", "Zarr", "RasterDataSources", 
    "Meshes", "Colors", "ColorSchemes"
]

println("--- Iniciando instalacion en Julia 1.11.3 ---")
Pkg.add(paquetes)

println("--- Vinculando RCall e IJulia ---")
Pkg.build("RCall")
Pkg.build("IJulia")

println("--- Forzando precompilacion total ---")
Pkg.precompile()
'
````

Este protocolo asegura que `JuliaConnectoR` en R opere sin requerir manipulaciones manuales en los binarios de la carpeta de artefactos.
