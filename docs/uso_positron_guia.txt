5) chequear que contenedores están corriendo


a) 
docker ps

Muestra únicamente los contenedores que están activos actualmente. Permite verificar que contenedor_postgis_unal y el contenedor instanciado por Positron estén operando.

b)
docker ps -a

Lista todos los contenedores en el sistema, incluyendo aquellos que se han detenido por errores de configuración o salida del proceso principal. Es fundamental para identificar si el contenedor de análisis falló durante el arranque.

c)
docker compose ps


Este comando debe ejecutarse desde la raíz del directorio final/. Proporciona una visión orientada a servicios, mostrando el estado de los contenedores definidos en el archivo docker-compose.yml y su mapeo de puertos.

d) 
docker ps --format "table {{.Names}}\t{{.Status}}\t{{.Ports}}"


Genera una salida tabulada y simplificada, facilitando la lectura de los nombres asignados y la persistencia de los puertos mapeados (e.g., 5434 para PostGIS).

e)

docker network inspect final_default

8) Mensaje de error en el intérprete de python

Análisis del error del intérprete de Python
El error registrado en el log (/bin/sh: 1: C:Program: not found) se origina porque la extensión de Python en Positron intenta validar rutas del sistema host (Windows) dentro del entorno Linux del contenedor. Esto es un remanente de la configuración de QGIS en la máquina local que la extensión intenta "descubrir" automáticamente.

Para corregir esto y evitar que Positron intente usar el ejecutable de Windows:

Abrir los ajustes de Positron (Ctrl + ,).

Seleccionar la pestaña Remote [Dev Container].

Buscar la opción Python: Default Interpreter Path.

Establecer explícitamente la ruta /usr/bin/python3.
